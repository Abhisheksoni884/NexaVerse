

# AI-Powered Enterprise Knowledge Assistant (Azure RAG)

Build a **production-ready Enterprise Knowledge Assistant** using **React (TypeScript)**, **FastAPI (Python)**, and **Microsoft Azure AI Services**.

The application should allow employees to upload enterprise documents (PDF, DOCX, images), ask natural language questions, and receive accurate, citation-backed answers using a **Retrieval-Augmented Generation (RAG)** pipeline.

## Core Features

### 1. Authentication & Role-Based Access Control (RBAC)

* Authenticate users using **Azure Entra ID** (or JWT-based simulated authentication for the prototype).
* Support three roles:

  * **Admin**
  * **Analyst**
  * **Viewer**
* Enforce RBAC at:

  * UI level
  * API level
  * Search level (restricted documents should never reach the LLM).

---

### 2. Document Upload & Processing

Allow users to:

* Upload PDF, DOCX, JPEG, PNG, and TIFF files.
* Validate file type and size.
* Store files in **Azure Blob Storage**.
* Extract text, tables, key-value pairs, and OCR using **Azure AI Document Intelligence**.
* Chunk documents semantically with metadata such as:

  * document name
  * page number
  * section
  * category
  * uploader
* Generate embeddings using **Azure OpenAI**.
* Index chunks in **Azure AI Search** with hybrid (vector + keyword) search.
* Display real-time processing status:
  Upload → Extracting → Chunking → Indexing → Ready.

---

### 3. Conversational RAG Chat

Create a modern chat interface that supports:

* Streaming responses (SSE)
* Markdown rendering
* Threaded conversations
* Multi-turn context
* Follow-up questions
* Conversation history management within token limits

The RAG pipeline should:

1. Generate query embeddings.
2. Retrieve top-k relevant chunks using hybrid search.
3. Apply RBAC filters.
4. Build grounded prompts.
5. Generate responses using Azure OpenAI GPT-4o.

---

### 4. Citations

Every answer must include:

* Inline citations
* Source panel
* Document name
* Page number
* Excerpts

Users should only see citations from documents they are authorized to access.

---

### 5. Content Safety

Use **Azure AI Content Safety** to:

* Validate user inputs.
* Validate AI outputs.
* Block unsafe content with a friendly fallback message.

---

### 6. Audit Trail

Log every important event in **Azure Cosmos DB**, including:

* Login/logout
* Document upload/delete
* Chat queries
* Content safety violations
* RBAC access denials
* Admin actions

Each log should contain:

* timestamp
* user
* role
* action
* resource
* IP/session (if available)
* token usage

Create an **Admin Audit Dashboard** with:

* filtering
* pagination
* sorting
* CSV/JSON export.

---

### 7. Token Usage Tracking

Track Azure OpenAI usage for:

* Chat completions
* Embeddings

Store:

* prompt tokens
* completion tokens
* total tokens

Maintain:

* daily
* weekly
* monthly
* all-time usage

Provide:

**User Dashboard**

* personal usage
* recent queries
* token breakdown
* quota progress (optional)

**Admin Dashboard**

* all-user analytics
* leaderboard
* trend charts
* cost estimation
* export reports.

---

### 8. Document Management

Build a document library showing:

* upload status
* page count
* chunk count
* upload date
* category
* uploader

Admins can:

* delete documents
* change document category

Non-admin users only see documents they are permitted to access.
